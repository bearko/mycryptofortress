## 1. 概要

[SPEC-001](./SPEC-001-mvp-stage1.md) で立ち上げた Stage1-1 MVP のバトル画面を、原作 Unity プロジェクト `MCHTowerDefence` の挙動に近づけて遊べる粒度まで仕上げる。具体的には:

1. **ピクセルアート描画**: ヒーロー / 敵スプライトをニアレストネイバー法で拡大し、ぼやけを解消する。
2. **タイルベース攻撃範囲**: ヒーローの攻撃範囲を「円」ではなく「ヒーロータイルからの相対オフセットの集合」で定義する（直線・扇形・コーンなど形状の差別化が可能になる）。
3. **配置時の向き選択**: ヒーロー設置直後、確定前にカーソル位置で向き（上下左右の 4 方向）を選び、攻撃範囲タイルがその向きに回転する。スプライト自体の向きは変えず、攻撃範囲だけ変わる。
4. **未移植の Unity 機能**: 右クリックで売却、左クリックで簡易ステータスパネル、再生速度トグル（1.0× / 2.0×）。

## 2. 背景と動機

- SPEC-001 は最低限のループに集中し、攻撃範囲を「円」「直線距離」で簡略化した。これだとヒーロー間の差別化が弱く、配置の戦略性が薄い。
- Unity 版では攻撃範囲がスプライトとして個別定義されており（`HealRangeTile`, `AttackRange` 等のプレハブ）、ヒーローごとに異なる「形」を持っていた。タイル単位での表現に揃える方が原作再現性も高い。
- 配置時の向き選択は Unity 版 `Player.cs` の `HeroSummonFlag` フローにあり、配置 UX として重要。再現する。
- 描画は Phaser の `pixelArt: true` で NEAREST が効くはずだが、`setDisplaySize` の非整数倍スケールでぼやけが出ているため明示的に整数倍 + テクスチャ単位 NEAREST 設定で固める。

## 3. 目標と非目標

### 目標

- ヒーロー / 敵のスプライトがぼやけず、ピクセルアートとして見える。
- 各ヒーローが固有の攻撃範囲パターン（タイル集合）を持ち、画面上で「どのタイルが攻撃対象か」が一目でわかる。
- 配置直後に向きを選べる（カーソル方向で 4 方向にスナップ → クリックで確定）。
- 配置済みヒーローを右クリックで売却（コストの 1/2 を CE に返金）。
- 配置済みヒーローを左クリックでステータスパネルが開く（パネル表示中はゲーム速度を 0.1× に低下＝ほぼ停止）。
- HUD に再生速度トグル（1.0× / 2.0×）。

### 非目標

- ヒーローのレベルアップ・スキル発動・回復ヒーロー / デバフヒーロー（別 SPEC）。
- 敵の攻撃 / 状態異常 / 多種類化（別 SPEC）。
- セーブデータ・複数ステージ・BGM 切替・カットイン演出（別 SPEC）。
- ダメージ式の見直し（別 SPEC、Unity 版の「高防御 = 多ダメ」逆転は維持）。
- ヒーローへの被ダメージ表現（敵が攻撃しないため不要）。

## 4. スコープ

### 含む

- `web/src/game/types.ts`: `Direction`, 攻撃パターンの型と回転ヘルパ追加。
- `web/src/game/heroData.ts`: 各ヒーローの `attackPattern` と簡単な向きデフォルトの定義。
- `web/src/scenes/BootScene.ts`: 全テクスチャに NEAREST フィルタを明示適用。
- `web/src/scenes/StageScene.ts`:
  - スプライトサイズを TILE_SIZE と同じ 64 px（整数倍スケール）にする。
  - 攻撃範囲表示をタイルベースに変更（半透明の四角タイル）。
  - 配置フローを 2 段階化: 「タイル選択 → 向き選択 → 確定」。
  - 右クリック売却。
  - 左クリックでステータスパネル（モーダル）。
  - HUD に再生速度トグル。
- 単位テスト: 攻撃パターンの 4 方向回転、カーソル方向 → 4 方向スナップ。

### 含まない

- 既存ダメージ式の変更（SPEC-001 を維持）。
- 敵の AI 強化、追加敵、追加 Wave。

## 5. 契約と挙動

### 5.1 攻撃パターン（タイルベース）

```ts
// 右向きを正規形とする。各エントリはヒーロータイルからのオフセット (col, row) で、
// 攻撃が届くタイルを示す。
type AttackPattern = TilePos[];
```

回転規則（`rotatePattern(pattern, dir)`）:

- **right**（正規形）: そのまま
- **down**: `(c, r)` → `(-r, c)`
- **left**: `(c, r)` → `(-c, -r)`
- **up**: `(c, r)` → `(r, -c)`

各ヒーローのパターン（暫定値、PR で調整可）:

| Hero | 向きの初期値 | 正規形パターン（右向き） | 図 |
|------|-------------|------------------------|-----|
| #1001 (INT) | right | `[(1,0),(2,0),(3,0),(4,0)]` | 4 タイル直線 |
| #3009 (PHY) | right | `[(1,-1),(1,0),(1,1),(2,-1),(2,0),(2,1),(3,0)]` | 3 タイル幅 + 先端 1 |
| #4003 (PHY) | right | `[(1,-1),(1,0),(1,1)]` | 1 タイル前 3 マス（縦広め） |

### 5.2 配置フロー

1. ヒーローパレットでヒーローをクリック（CE 不足なら拒否）→ ゴーストがカーソル追従。
2. 配置可能タイル（path 以外、空き）をクリック → ヒーローが半透明で配置され、CE は仮確保（まだ消費せず保持）。
3. **向き選択フェーズ開始**: カーソル位置とヒーロータイルの差分から 4 方向にスナップして向きを更新、攻撃範囲タイルが動的に再描画される。
4. 任意の位置を再クリック → 確定。CE を消費、ヒーロー不透明化、攻撃ループ開始。
5. 配置中にパレットを再クリック / 右クリックでキャンセル可能（CE 仮確保を解放）。

### 5.3 攻撃範囲表示

- 配置中: 半透明の黄色タイル（透過度 0.25 程度、枠あり）。
- 配置済み: 同じ色を透過度 0.10 程度に下げて常時表示。
- ヒーローを左クリックでステータスパネルを開いた間は、当該ヒーローの範囲タイルを強調（透過度 0.35）。

### 5.4 ターゲティング

- 各攻撃間隔で、ヒーローは `heroTile + rotatedPattern` のいずれかのタイルにいる敵のうち、最も近い 1 体に射撃。
- 敵タイルは `pixelToTile(enemy.sprite.x, enemy.sprite.y)` で求める。
- 弾は SPEC-001 と同じ。射程外に出た敵は対象外（弾は target に追従するが、target が消えたら弾も消滅）。

### 5.5 売却

- 配置済みヒーロー上で **右クリック**: 売却。CE += `ceil(cost / 2)`。確認ダイアログなし（Unity 準拠）。
- 売却対象が選択中なら選択解除。

### 5.6 ステータスパネル

- 配置済みヒーロー上で **左クリック**: モーダルパネルを開く。
  - 表示項目: 名前、画像、HP / 最大 HP、PHY、INT、AGI、攻撃間隔、攻撃パターン（小サムネ）、コスト。
  - 表示中はゲーム速度を 0.1× に低下（実質停止）。
  - パネル背景の暗幕クリックまたは「閉じる」ボタンで閉じる → 速度を直前の値に戻す。

### 5.7 再生速度トグル

- HUD 右下に「1.0×」/「2.0×」ボタン。クリックで切替。
- 内部の `playSpeed` を CE 蓄積、敵移動、弾移動、攻撃間隔のすべてに反映する。

### 5.8 描画（ニアレストネイバー）

- `BootScene` で全画像テクスチャに `texture.setFilter(Phaser.Textures.FilterMode.NEAREST)` を明示適用。
- スプライト表示サイズは **TILE_SIZE = 64 px** に揃える（ヒーロー / 敵とも 64×64）。素材が 64×64 なら 1:1、それ以外でも NEAREST により拡大ぼやけが出ないことを期待。
- HTML 側の `image-rendering: pixelated` は維持。

## 6. 非機能要件

- 性能: SPEC-001 と同じ 60fps を維持。攻撃範囲タイルは最大でも 1 ヒーローあたり 10 タイル程度なので追加負荷は無視できる。
- アクセシビリティ: 既存 SPEC-001 と同じ（MVP では未対応）。

## 7. マージ前確認事項（HITL）

- ゲーム数値の調整あり: 攻撃パターンを「円」から「タイル集合」に変更しているため、各ヒーローの実効火力が変わる。PR 本文で初期パターンの妥当性を併記。

## 8. 受け入れ基準

- [ ] ヒーロー / 敵スプライトが NEAREST で拡大され、ピクセルアートのドットが視認できる。
- [ ] 各ヒーローの攻撃範囲が円ではなくタイル集合で表示される。
- [ ] 配置時、カーソル方向で攻撃範囲が回転し、再クリックで確定する。
- [ ] 確定後の右クリックでヒーローが売却され、CE が `ceil(cost/2)` 戻る。
- [ ] 確定後の左クリックでステータスパネルが開き、ゲームが実質停止する。
- [ ] HUD のトグルで 1.0× / 2.0× 切替が機能する（敵移動・CE 蓄積も連動）。
- [ ] `npm run typecheck` / `npm test` / `npm run build` が緑。
- [ ] `rotatePattern` と `directionFromDelta` をユニットテストで網羅。
