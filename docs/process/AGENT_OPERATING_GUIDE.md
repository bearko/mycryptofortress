# エージェント運用ガイド（AI / 自動化）

人間とエージェントが同じ憲章と仕様の下で協働するための境界。

## 1. 入力として必須に近いもの

- 対象 SPEC ID または文書へのリンク
- 意図（達成事項と非目標）
- 制約（環境、触ってはいけない領域）

## 2. 自律的に進めてよい範囲（広め）

- SPEC と憲章に沿った実装・リファクタ・テスト・型・リント
- ドキュメントの誤字補正・明らかな抜けの追加
- 依存関係の minor / patch 更新（major は PR 本文で論点化）
- ブランチ作成・コミット・push（保護ブランチへの直 push 以外）
- PR の作成・編集・コメント

## 3. 止まるべき範囲（最小化）

技術的に不可避な箇所のみ:

- 秘密情報の生成・取得・コミット
- 破壊的シェル操作（`rm -rf`, 保護ブランチへの `git push --force`, `gh repo delete`）
- 外部 API への本番書き込み

それ以外は **PR を作って人間レビューに委ねる**。

## 4. PR で論点を提示する作法

[HUMAN_IN_THE_LOOP.md](./HUMAN_IN_THE_LOOP.md) の論点に該当する変更を含む場合:

1. 作業は完走させる（テスト追加・型/ビルド成功まで）
2. PR 本文の「マージ前確認事項（HITL）」セクションに該当項目をチェック
3. 該当項目の理由・代替案・推奨選択を箇条書きで併記
4. 人間レビュアにマージ可否を委ねる

## 5. 出力の品質

- 変更理由を PR 本文またはコミット本文に残す
- 不確実な点は仮定として列挙し、検証方法を添える
- スクリーンショット・ローカル実行ログなどの証跡を貼る
