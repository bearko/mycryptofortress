# テスト戦略

ゲーム特有の挙動（描画・入力）は完全な自動化が難しいため、**ユニットテスト + 手動確認証跡** の二段構えとする。

## 1. ユニットテスト（Vitest 想定）

優先的にカバーするロジック:

- ダメージ計算（`calculatePhyDamage`, `calculateIntDamage`, ヒール量）
- Wave スケジューラ（`time` を進めると正しい敵が出てくる）
- Route 上の位置補間（`pointIndex` の進行とゴール検知）
- CE 蓄積モデル（時間経過で 0 → 100 になる）

未対応: Phaser のシーン・スプライト描画はユニットテスト対象外。

## 2. 手動確認

PR で確認する内容（Stage1-1 想定）:

- [ ] タイトル → ステージへの遷移
- [ ] CE が時間で蓄積し、コストを払ってヒーローを設置できる
- [ ] ヒーローが射程内の敵に自動攻撃して倒せる
- [ ] 敵がゴールに到達するとプレイヤー HP が減る
- [ ] 全敵を倒すとクリア、HP が 0 になると敗北画面

証跡として PR 本文にスクリーンショット 1 枚以上を貼る。

## 3. ビルドチェック

```bash
cd web
npm install
npm run typecheck
npm run build
```

PR では上記 3 つが緑であることを本文に明記する。
